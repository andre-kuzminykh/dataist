# Урок 1 — Loop Engineering · Спецификация карточек

Анимированные карточки‑слайды для Instagram (и других соцсетей) по уроку
**«Loop Engineering»**. Один HTML‑файл (`index.html`) = карусель из **8 квадратных
слайдов** 1080×1080. На каждом слайде есть **постоянная анимация** (что‑то крутится /
пульсирует / течёт), чтобы зритель «залипал».

Это живой контракт: код в `index.html` и автотесты в `tests/` должны соответствовать
этому документу. Менять контент слайда — значит обновить и спеку, и тест.

---

## 1. Формат и общие правила

| Параметр | Значение |
|---|---|
| Размер слайда | **1080 × 1080** (квадрат, IG feed/carousel) |
| Кол-во слайдов | **8** (`#card-1` … `#card-8`) |
| Шрифт | **JetBrains Mono** (Google Fonts) + fallback `ui-monospace, Menlo, monospace` |
| Обложка (слайд 1) | **тёмный** фон `#050505` |
| Слайды 2–8 | **белый / светлый** фон `#FAFAFA` |
| Логотип AIT | маленький, в **левом верхнем** углу каждого слайда |
| Иконки | **inline‑SVG** (без CDN) — крупные линейные, в стиле Lucide |
| Экспорт | каждый `.card` рендерится в `screenshots/slide-N.png` при `?export` |

### Палитра — строго ДВА цвета (минимализм)
Только **оранжевый + фиолетовый**, плюс нейтрали (белый, серый текст, тёмный фон обложки).
Никаких зелёного/синего/прочих — это проверяется тестом «палитра» в `tests/verify.mjs`.

| Токен | HEX | Роль |
|---|---|---|
| flame (оранжевый) | `#F97316` | акцент A, блок‑инсайт, логотип |
| plum (фиолетовый) | `#8854F3` | акцент B |
| нейтрали | `#FAFAFA` / `#050505` / серые | фон, текст, иконки среды |

Акценты на всех схемах **чередуются** orange↔purple. Старые токены `--green/--blue/--sky/--ink`
оставлены как алиасы на `--flame`/`--plum`, чтобы не плодить цвета. Геро‑элементы
(АГЕНТ, хаб Loop, блок МОДЕЛЬ) — градиент `plum → flame`.

### Логотип
- Основной `src` — реальный логотип AIT:
  `https://raw.githubusercontent.com/andre-kuzminykh/andre-ait-page/main/866f2500-dd81-4d09-8c0f-2b55c25a3464-removalai-preview.png`
  (он же `https://github.com/.../...png?raw=true`).
- `onerror` → локальная копия `assets/ait-logo.png` — карточка никогда не покажет «битую»
  картинку (в т.ч. офлайн и на скриншотах) и всегда показывает настоящий логотип.

### Блок‑инсайт (слайды 2–8)
Внизу — оранжевая плашка с **иконкой лампочки** (пульсирует) и ключевой мыслью слайда.

## 2. Анимация — обязательна на каждом слайде

Базовая библиотека keyframes (в `index.html`): `spin`, `spin-rev`, `floatY`, `pulse`,
`breathe`, `dash` (поток пунктира), `shimmer` (градиент), `travel` (точка по кругу/линии),
`sweep` (последовательная подсветка), `blink`, `ping`, `draw`.

| # | Слайд | Главная постоянная анимация |
|---|---|---|
| 1 | Обложка | вложенные рамки «дышат» по очереди (sweep+pulse), за центром вращается конический блик, слева плывут иконки‑тайлы, стрелка‑подсказка пульсирует |
| 2 | Prompt | стрелка ПРОМТ→ОТВЕТ течёт вправо (chevrons), 4 строки промта подсвечиваются по кругу, пунктирная обратная связь течёт, «печатаются» строки ответа |
| 3 | Context | по 5 линиям‑коннекторам **текут точки вниз** в воронку (бесконечно), слои воронки пульсируют, исходные карточки подсвечиваются по очереди |
| 4 | Harness | **пунктирное кольцо вращается** вокруг агента, иконки внешней среды вращаются по орбите, лучи к инструментам пульсируют, агент «дышит» + ping‑кольца |
| 5 | Loop | **круговые стрелки вращаются** вокруг агента, по контуру бежит светящаяся точка, верхняя цепочка подсвечивается слева‑направо |
| 6 | Хороший loop | центральная круговая стрелка **вращается**, 6 карточек плавают со сдвигом, пунктир‑коннекторы текут к центру |
| 7 | Применение | по сетке 2×3 бежит «волна» подсветки рамок (sweep), карточки плавают со сдвигом, галочка дорисовывается |
| 8 | Практики | по списку из 6 пунктов сверху‑вниз бежит акцент‑подсветка (sweep), левые иконки пульсируют, нижний блок «дышит» |

`prefers-reduced-motion: reduce` — анимации останавливаются (доступность), но это не
дефолт.

## 3. Контент по слайдам (точный текст)

### Слайд 1 — Обложка (тёмная)
- Заголовок (2 строки): **Хватит самому / писать промты**
- Вложенная схема (снаружи внутрь), контуры чередуются оранжевый↔фиолетовый:
  - `Loop Engineering` — оранжевый контур, иконка **цикл**
  - `Harness Engineering` — фиолетовый, иконка **настройки**
  - `Context Engineering` — оранжевый, иконка **документ**
  - `Prompt Engineering` — фиолетовый, иконка **чат**
- Финальная фраза: **Теперь мы строим системы, которые промптят агентов вместо нас →**

### Слайд 2 — 1. Prompt Engineering
- Заголовок: **1. Prompt Engineering**
- Подзаголовок: **Учимся правильно просить модель**
- Левая карточка (фиолетовая) **ПРОМТ**, строки (чередуются цвета): **Роль · Формат · Ограничения · Примеры**
- Стрелка вправо → Правая карточка (оранжевая) **ОТВЕТ** (иконка сообщения с галочкой, серые линии)
- Под схемой: **Ручное управление моделью** (пунктирная обратная линия) и плашка **Один промт → один ответ**
- Инсайт: **Хороший промт улучшает ответ ИИ. Но хороший ответ — это еще не итоговый результат**

### Слайд 3 — 2. Context Engineering
- Заголовок: **2. Context Engineering**
- Подзаголовок: **Даем модели правильную информацию**
- 5 карточек‑источников (чередуются оранж/фиол): **Документы** (оранж), **Память** (фиол), **История** (оранж), **Правила** (фиол), **Данные** (оранж)
- Линии вниз → воронка **КОНТЕКСТ** (иконка слои) → блок **МОДЕЛЬ** (иконка чат)
- Инсайт: **Контекст должен не заполнять окно модели, а объяснять, как работает система**

### Слайд 4 — 3. Harness Engineering
- Заголовок: **3. Harness Engineering**
- Подзаголовок: **Строим среду вокруг агента**
- Центр: **АГЕНТ** (робот)
- 6 карточек вокруг: **Инструменты** (ключ), **Память** (мозг), **MCP** (штекер), **Разрешения** (щит), **Guardrails** (щит‑галка), **Логи** (документ)
- Пунктирное кольцо с иконками среды: **браузер, база данных, календарь, почта, GitHub, документ**
- Инсайт: **Модель без обертки даёт ответ. Модель с оберткой умеет действовать**

### Слайд 5 — 4. Loop Engineering
- Заголовок: **4. Loop Engineering**
- Подзаголовок: **Строим цикл, который делает это вместо нас**
- Верхняя цепочка: **Цель → Контекст → Действие → Наблюдение → Корректировка**
- Центр **АГЕНТ**; круговой цикл из 4 узлов: **Цель** (мишень, сверху), **Действие** (молния, справа), **Фидбек** (сообщение‑галка, снизу), **Память** (мозг, слева)
- Инсайт: **Loop — это не один запуск. Система действует, получает фидбек и повторяет цикл**

### Слайд 6 — Из чего состоит хороший loop
- Заголовок: **Из чего состоит хороший loop**
- Центр: **Loop** + круговая стрелка
- 6 карточек: **Автоматизация** — когда запускать · **Правила** — что делать дальше · **Навыки** — как выполнять задачи · **Коннекторы** — куда подключаться · **Агенты** — кто делает и кто проверяет · **Состояния** — что уже сделано и что дальше
- Инсайт: **Хороший loop не просто повторяет команду. Он знает, когда запускаться, что делать, чем проверять результат, где сохранить прогресс и когда остановиться**

### Слайд 7 — Для чего лучше использовать loop
- Заголовок: **Для чего лучше использовать loop**
- Подзаголовок: **Когда есть повторяемая работа и понятная проверка результата**
- Сетка 2×3: **Исправление ошибок** (жук+лупа), **Сортировка проблем** (серверы), **Обновление документации** (документ), **Обработка файлов** (папка), **Исследовательский проект** (лупа/график), **Проверка результатов** (галочка в круге)
- Инсайт: **Loop хорош там, где есть фидбек: тесты, логи, метрики, скриншоты или проверка человеком**

### Слайд 8 — Лучшие практики Loop Engineering
- Заголовок: **Лучшие практики Loop Engineering**
- Список из 6: **Задайте проверяемую цель** (мишень) · **Дайте правильный контекст** (сообщение) · **Делайте маленькие шаги** (ступеньки) · **Разделите создателя и проверяющего** (разветвление) · **Сохраняйте состояния** (база данных) · **Ограничьте автономность guardrails** (щит)
- Финальная фраза (оранжевый блок): **ИИ-агент может быть автономным ровно настолько, насколько система умеет проверять его работу**

## 4. Навигация / UX (на экране)
- Одна карточка по центру, масштабируется под экран (`--scale`), без обрезки.
- Управление: стрелки ◀ ▶, точки‑индикаторы, клавиши ← →, свайп, счётчик `01 / 08`.
- `?export` — все карточки видны в столбик в натуральную величину 1080×1080 (для скриншотов),
  навигация скрыта.

## 5. Критерии приёмки (проверяются в `tests/verify.mjs`)
1. Ровно **8** элементов `.card` с `id="card-1..8"`.
2. У каждого слайда есть **логотип** (`.logo img`) и он не «битый» (есть `onerror`‑fallback).
3. Слайд 1 — тёмный фон (близкий к `#050505`); слайды 2–8 — светлый (близкий к `#FAFAFA`).
4. На **каждом** слайде есть хотя бы один элемент с активной CSS‑анимацией
   (`getComputedStyle(...).animationName !== 'none'` и `animationPlayState === 'running'`).
5. Слайды 2–8 содержат **блок‑инсайт** с иконкой лампочки.
6. Применён шрифт **JetBrains Mono** (`document.fonts.check`).
7. Ключевые фразы каждого слайда присутствуют в DOM (см. список текста выше).
8. `screenshots/slide-1..8.png` генерируются (1080×1080; `SHOT_SCALE=2` → 2160×2160).
9. Палитра чистая: в `index.html` нет посторонних цветов (green/blue/sky) — только оранжевый+фиолетовый.
